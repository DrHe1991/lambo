## 1. 经济模型总览

### 核心设计原则

| 原则 | 说明 |
|------|------|
| **零平台补贴** | 平台不印钱，所有收益来自用户支付 |
| **创作者优先** | 80% 收入直接给创作者 |
| **质量导向** | 高质量内容获得更多曝光和收益 |
| **新人友好** | 新用户 7 天免费体验期 |
| **恶意无利可图** | 刷量、互刷、垃圾内容必亏 |

### 资金流向图

```
用户钱包 
    │
    ├──→ 发帖/评论/点赞 (支付 sat)
    │           │
    │           ├── 80% ──→ 内容创作者 (直接收入)
    │           │
    │           └── 20% ──→ 平台收入池
    │                           │
    │                           └── 100% ──→ 质量补贴 (每周分发)
    │                                            │
    │                                            └──→ 被低估的高质量内容作者
    │
    └──← 收入 (创作者分成 + 质量补贴)
```

## 2. 行为成本体系

### 基础成本 (单位: sat)

| 行为 | 成本 | 说明 |
|------|------|------|
| 发帖 | 50 | 降低创作门槛 |
| 问答 | 100 | 鼓励深度问答 |
| 评论 | 20 | 鼓励互动 |
| 回复 | 10 | 低门槛讨论 |
| **点赞** | **20** | 核心价值行为 |
| 评论点赞 | 10 | 次要互动 |

### 新用户免费额度 (前 7 天)

| 行为 | 每日免费次数 |
|------|-------------|
| 发帖 | 1 次 |
| 评论 | 5 次 |
| 点赞 | 10 次 |

> 第 8 天起，所有行为需付费

### 成本动态调整

实际成本 = 基础成本 × (1 + Spam Index)

- **Spam Index**: 0.0 ~ 1.0，反映平台近期垃圾内容比例
- 垃圾内容越多，所有行为成本越高（集体惩罚机制）

## 3. 收入分配机制

### 3.1 直接分成 (实时)

当用户点赞时：
- **80%** → 内容创作者（立即到账）
- **20%** → 平台收入池

**回本计算**：
- 发帖成本 50 sat，每个赞给作者 16 sat
- **只需 4 个赞即可回本**

### 3.2 质量补贴 (每周)

平台累积收入 100% 用于补贴「被低估的高质量内容」。

**补贴对象筛选条件**：

| 条件 | 阈值 | 目的 |
|------|------|------|
| 最低点赞数 | ≥ 2 | 过滤无人问津的内容 |
| 点赞排名 | 低于 70 分位数 | 只补贴曝光不足的内容 |
| 质量密度 | ≥ 0.35 | 确保内容有质量信号 |
| 作者 Risk | < 100 | 排除高风险作者 |
| 非 Cabal 成员 | ✓ | 排除互刷组织成员 |

**质量密度计算**：

```
quality_density = 
    点赞者平均Trust × 0.35 +
    跨圈点赞比例 × 0.25 +
    评论/点赞比 × 0.15 +
    点赞者多样性 × 0.15 +
    新作者奖励 × 0.10
```

**补贴分配**：按质量密度比例分配补贴池

### 3.3 声誉奖励

收到补贴时，作者还会获得 Trust 分数提升：
- 普通补贴：Creator +5~15
- 质量密度前 10%：额外 Creator +15~30

## 4. Trust 信任系统

### 4.1 四维声誉模型

| 维度 | 初始值 | 说明 |
|------|--------|------|
| **Creator** | 150 | 创作能力，无上限 |
| **Curator** | 150 | 策展眼光（点赞质量） |
| **Juror** | 300 | 陪审准确度 |
| **Risk** | 30 | 风险评分（越低越好） |

### 4.2 Trust Score 公式

```
TrustScore = Creator × 0.6 + Curator × 0.3 + Juror_bonus - Risk_penalty

其中：
- Juror_bonus = max(0, (Juror - 300) × 0.1)
- Risk_penalty = 
    if risk ≤ 50:  risk × 0.5
    if risk ≤ 100: 25 + (risk - 50) × 2
    if risk > 100: 125 + (risk - 100) × 5  # 严重惩罚
```

### 4.3 Trust 等级与阈值

| 等级 | 分数范围 | 目标占比 | 权益 |
|------|----------|----------|------|
| 🤍 White | 0 - 150 | ~72% | 基础权限 |
| 💚 Green | 151 - 200 | ~15% | 点赞权重 ×1.0 |
| 💙 Blue | 201 - 300 | ~8% | 点赞权重 ×2.0 |
| 💜 Purple | 301 - 450 | ~3.5% | 点赞权重 ×3.5 |
| 🧡 Orange | 451+ | ~1% | 点赞权重 ×6.0 |

### 4.4 声誉增长事件

**实时生效（每次行为）**：

| 事件 | 维度 | 变化 |
|------|------|------|
| 发帖（非违规） | Creator | +1 ~ +3 |
| 点赞 | Curator | +0.2 ~ +0.6 |
| 评论 | Curator | +0.3 ~ +0.8 |

**周结算（质量补贴）**：

| 事件 | 维度 | 变化 |
|------|------|------|
| 收到补贴 | Creator | +5 ~ +15 |
| 质量密度前 10% | Creator | +15 ~ +30 |

**惩罚（不受等级递减影响）**：

| 事件 | 维度 | 变化 |
|------|------|------|
| 内容违规 | Creator | -20 ~ -60 |
| 点赞违规内容 | Curator | -2 ~ -8 |
| Cabal 主犯 | Risk | +150 ~ +500 |

### 4.5 等级递减机制

**设计目的**：防止头部固化，加速新人成长

| 等级 | 声誉奖励倍率 |
|------|-------------|
| White | 100% |
| Green | 70% |
| Blue | 50% |
| Purple | 30% |
| Orange | 15% |

> 惩罚不递减，始终全额扣除

## 5. 内容推荐与曝光

### 5.1 推断质量 (Inferred Quality)

基于可观测信号推断内容质量（非上帝视角）：

```
inferred_quality = 
    engagement_score × 0.35 +     # 互动密度
    source_quality × 0.35 +       # 点赞者质量
    author_factor × 0.20 +        # 作者信誉
    (1 - negative_signals) × 0.10 # 负面信号
```

### 5.2 曝光权重 (Exposure Weight)

决定内容被用户看到的概率：

```
exposure = inferred_quality × time_decay × engagement_boost

其中：
- time_decay: 时间衰减（半衰期 3-11 天，质量越高衰减越慢）
- engagement_boost: 早期互动加成（前 24 小时有赞 = 更长曝光）
```

### 5.3 热门内容递减

防止单一爆款垄断曝光：

```
diminishing_factor = log(likes + 1) / (likes + 1)
adjusted_exposure = exposure × diminishing_factor
```

## 6. 点赞权重计算

点赞的价值不是固定的，而是多因素加权：

```
Like.weight = W_trust × N_novelty × S_source × CE_entropy × Cross_circle_mult × Cabal_penalty
```

### 各因子说明

| 因子 | 说明 | 取值范围 |
|------|------|----------|
| **W_trust** | 点赞者 Trust 等级 | 0.5 ~ 6.0 |
| **N_novelty** | 与作者的互动新鲜度 | 0.05 ~ 1.0 |
| **S_source** | 陌生人 vs 关注者 | 0.15 / 1.0 |
| **CE_entropy** | 共识多样性 | 0.02 ~ 10.0 |
| **Cross_circle** | 跨圈点赞加成 | 1.0 / 1.5 |
| **Cabal_penalty** | Cabal 成员惩罚 | 0.3 / 1.0 |

### N_novelty 详细规则

| 过去 30 天互动次数 | N_novelty |
|-------------------|-----------|
| 0 次（首次） | 1.00 |
| 1-3 次 | 0.60 |
| 4-10 次 | 0.30 |
| 11-30 次 | 0.12 |
| 30+ 次 | 0.05 |

> 频繁互动的朋友点赞价值递减

## 7. 反作弊机制

### 7.1 Cabal (互刷组织) 检测

**触发条件**：
- 组内互动 / 组外互动 > 3
- 平均组内互动量 > 50 次/人
- 成员 Risk 分数 > 80

**惩罚措施**：

| 惩罚 | 说明 |
|------|------|
| Risk +150~500 | 主犯直接拉黑 |
| Creator 扣除 | 500~1500 分 |
| 资产没收 | 30%~80% 余额 |
| 点赞权重 ×0.3 | 持续 30 天 |
| 取消补贴资格 | 永久 |

### 7.2 渐进式怀疑系统

在 Cabal 被正式检测之前，异常行为已经开始降权：

```
suspicion_level = min(1.0, internal_ratio / 5)
weight_penalty = suspicion_level × 0.7  # 最高降权 70%
```

### 7.3 圈子贡献限制

每人每天最多给「圈内人」贡献 100 点发现分权重：

```
每日圈子贡献上限 = 100 DS
超出部分的点赞权重 = 0
```

### 7.4 共识熵 (CE_entropy)

| 点赞来源 | CE 权重 | 说明 |
|----------|---------|------|
| Cabal 互刷 | 0.02 | 几乎无效 |
| 高频好友 | 0.15 | 大幅降权 |
| 同频道 | 1.0 | 正常 |
| 跨频道 | 5.0 | 价值更高 |
| 跨地区 | 10.0 | 价值最高 |

| 接收方 | 比例 |
|--------|------|
| 举报者 | 35% |
| 陪审团 | 25% |

- 如果是真的：收益 ×1.2

## 9. Post Boost 付费曝光

### 9.1 概述

广告商可以花钱购买额外曝光，这是平台最大的收入来源之一。

### 9.2 机制设计

```
Boost 成本 → 曝光加成
├── 50% → 平台收入池 (用于质量补贴)
└── 50% → 平台运营收入

曝光公式:
boosted_exposure = base_exposure × min(5.0, 1.0 + boost_remaining)
```

### 9.3 参数

| 参数 | 值 | 说明 |
|------|-----|------|
| `BOOST_SAT_PER_POINT` | 100 | 100 sat = 1 曝光点 |
| `BOOST_MAX_MULTIPLIER` | 5.0 | 最大 5x 曝光加成 |
| `BOOST_DAILY_DECAY` | 0.7 | 每天衰减 30% |
| `BOOST_MIN_AMOUNT` | 1,000 sat | 最低 boost 金额 (~$0.68) |
| `BOOST_POOL_SHARE` | 50% | 进入补贴池的比例 |

### 9.4 Boost 衰减曲线

```
Day 0: 100% 效果
Day 1: 70% 效果
Day 2: 49% 效果
Day 3: 34% 效果
Day 5: 17% 效果
Day 7: 8% 效果 (几乎无效)
```

> 设计理由：Boost 是临时曝光加成，不应永久影响内容排名

### 9.5 广告商 (Advertiser) 用户类型

| 属性 | 值 | 说明 |
|------|-----|------|
| 占比 | 1% | 平台最大金主 |
| 初始余额 | $500-2000 | 高充值入场 |
| 月充值 | $200-1000 | 持续投入 |
| Boost 概率 | 80% | 大部分帖子会 Boost |
| Boost 金额 | $10-100 | 每帖 Boost 预算 |
| 内容质量 | 0.50 | 商业内容，中等质量 |
| 违规率 | 10% | 低违规（商业合规） |

## 10. 悬赏问答系统

### 10.1 概述

提问者支付悬赏金激励高质量回答，社区投票决定优胜者。

### 10.2 流程

```
提问阶段 (Day 0)
    │
    ├── 提问者支付: C_QUESTION (100) + 悬赏金 (500-1.5M sat)
    │   ├── 90% → 托管池 (给回答者)
    │   └── 10% → 平台收入
    │
回答阶段 (Day 1-7)
    │
    ├── 回答者支付: C_ANSWER (50 sat)
    ├── 每人限 1 个回答
    │
投票阶段 (Day 8-10)
    │
    ├── 社区对回答点赞投票
    ├── 投票权重 = 标准 Like.weight
    │
结算阶段 (Day 11)
    │
    ├── 悬赏分配:
    │   ├── 30% BASE_POOL → 平均分给所有有效回答
    │   └── 70% PERF_POOL → 按投票权重分配
    │
    ├── 声誉奖励:
    │   ├── 第1名: Creator +20~40
    │   ├── 第2名: Creator +10~20
    │   └── 第3名: Creator +5~10
    │
持续收益 (Day 11+)
    │
    └── 回答作为普通内容存在，后续点赞 → 回答者获得 80% 分成
```

### 10.3 参数

| 参数 | 值 | 说明 |
|------|-----|------|
| `BOUNTY_MIN` | 500 sat | 最低悬赏 (~$0.34) |
| `BOUNTY_MAX` | 1,500,000 sat | 最高悬赏 (~$1000) |
| `BOUNTY_PLATFORM_FEE` | 10% | 平台抽成 |
| `BOUNTY_ANSWER_WINDOW` | 7 天 | 回答窗口期 |
| `BOUNTY_VOTE_WINDOW` | 3 天 | 投票期 |
| `BOUNTY_BASE_POOL_RATIO` | 30% | 基础池比例 |
| `BOUNTY_PERF_POOL_RATIO` | 70% | 绩效池比例 |

### 10.4 防作弊机制

| 作弊场景 | 防御措施 |
|---------|---------|
| **自问自答** | 提问者不能回答自己的问题 |
| **Cabal 互刷** | CE_entropy = 0.02，投票几乎无效 |
| **马甲小号** | 新用户 (Trust < 150) 投票权重 ×0.5 |
| **恶意举报优质回答** | 判无罪则举报者扣 Risk |
| **提问者跑路** | 悬赏金托管，自动结算 |
| **无人回答** | 窗口期满无回答，90% 退还提问者 |

### 10.5 提问者选择权（可选功能）

提问者可在投票期后 24 小时内"认可"最佳答案：
- 认可权重 = 所有投票权重的 **30%**
- 如不选择，完全由社区投票决定

> 设计理由：提问者最了解自己的需求，但不能完全控制结果

### 10.6 回答质量门槛

| 条件 | 影响 |
|------|------|
| 回答长度 < 50 字符 | 权重 ×0.3 |
| 被 AI 检测为低质 | 权重 ×0.5 |
| 被成功举报违规 | **取消参选资格** |

### 10.7 关于「经久不衰」的回答

悬赏问答的回答在结算后变成普通内容：

```
悬赏奖励 (一次性)     持续收益 (长期)
      │                    │
      ▼                    ▼
  结算时分配          后续用户点赞
  (提问者的钱)        (新用户的钱)
```

- **悬赏是一次性的**：提问者的钱在结算时分完
- **好回答持续赚钱**：后续点赞按标准 80% 分成给回答者
- **无需提问者持续付费**：提问者只付一次悬赏

> 这意味着：一个精彩的回答可以成为作者的「数字资产」，持续数月甚至数年产生收益
