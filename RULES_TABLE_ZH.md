## 平台经济与治理规则表（严格删除垃圾版）

本文件定义平台内部“聪账本”（off-chain ledger）的**行为费用/质押/锁定/退回/罚没/分配/信誉变动**规则，用于实现“发言有成本、垃圾必亏钱、正常用户可退押金”的机制。

> 关键取向：**严格删除垃圾**。在不确定时更倾向于删、并通过上诉机制给“被误伤的优质内容”二次救济。

---

## 0. 术语与全局约束

- **sat（聪）**：平台内部统一结算单位（1 BTC = 100,000,000 sat）。支持 BTC/闪电/稳定币入金，但入账统一记为 sat 余额。
- **内部余额**：记在数据库的 sat 余额；提现时由平台发起链上/闪电转账（可收取网络费/提现费）。
- **锁定期**：除特别说明外，所有质押锁定期统一为 **24h**。
- **罚没（Slash）**：被判定违规/作弊/缺席等，会扣除质押的一部分或全部。
- **严格删垃圾策略**：
  - 案件裁决阈值偏向“更容易成立违规”
  - 上诉推翻更难（需要更强共识）
  - 陪审员缺席/乱投票惩罚更重

## 1. 动态费率（反垃圾难度）与基础参数

### 1.1 垃圾压力指数 `SI`（Spam Index）

平台每小时（或每天）更新一次 `SI ∈ [0, 1]`，用于动态调整质押/费用。

建议 `SI` 由以下信号加权得到（可从 0 开始逐步加信号）：
- `challenge_success_rate_24h`：过去 24h challenge 违规成立率
- `new_account_ratio_24h`：过去 24h 新账号发帖占比
- `duplicate_content_rate_24h`：重复/模板化内容占比
- `user_hide_block_rate_24h`：隐藏/拉黑/负反馈率
- `spam_cluster_score_24h`：图关系/设备/IP/支付路径异常聚类得分

### 1.2 动态倍率 `M(SI)`

统一倍率：

- `M = 1 + 3*SI`
  - `SI=0` → `M=1`（正常期）
  - `SI=1` → `M=4`（垃圾攻击期）

### 1.3 信誉折扣 `K(trust)`

用户综合信任分 `TrustScore ∈ [0, 1000]`。

- `K = clamp(1.4 - TrustScore/1250, 0.6, 1.4)`
  - 高信誉更便宜（最低 0.6 倍）
  - 低信誉更贵（最高 1.4 倍）

### 1.4 核心基础数值（可调默认）

> 下面是“可落地的默认区间”，上线后可根据 SI、留存、误伤率迭代。

- `D_post_base`：发帖基础质押 = **300 sat**
- `D_question_base`：提问基础质押 = **500 sat**
- `D_answer_base`：回答基础质押 = **400 sat**
- `D_comment_base`：评论基础质押 = **200 sat**
- `D_vote_base`：投票/点赞基础质押 = **30 sat**
- `F_challenge_base`：发起 challenge 基础费用（违规成立时退还，不成立时不退）= **100 sat**
- `B_challenge_base`：挑战者保证金（防滥诉，违规成立全退）= **500 sat**
- `B_juror_base`：陪审员保证金（缺席/乱投票罚没）= **300 sat**
- `N_light`：轻案陪审团规模 = **9**
- `N_heavy`：重案陪审团规模 = **15**
- `Quorum`：法定揭示人数比例 = **≥ 2/3**
- `T_vote_commit`：投票提交窗口 = **2h**
- `T_vote_reveal`：投票揭示窗口 = **4h**

实际执行时所有 `D_*`/`F_*`/`B_*` 乘以：

- `amount = base * M(SI) * K(TrustScore)`

## 2. 信誉系统（核心输入/输出）

### 2.1 四维子分（建议都映射到 0–1000）

- `CreatorScore`：内容生产信誉（你发的内容质量/违规历史）
- `CuratorScore`：点赞/投票判断力（你支持的内容是否最终被证明是好内容）
- `JurorScore`：陪审准确性与可靠性（投票是否与最终裁决一致、是否缺席、是否被上诉推翻）
- `RiskScore`：账户风险分（盗号/灰产/异常资金流风险；越高越危险）

**新用户初始值**：
- `CreatorScore = 500`，`CuratorScore = 500`，`JurorScore = 500`，`RiskScore = 0`
- 新用户初始 `TrustScore ≈ 550`（见下方公式）。这意味着新用户无法立即进入陪审池（需 ≥ 600），但经过一段时间的正常参与即可达标。

综合信任分（用于费用折扣、提现额度、陪审入池等）：

- `TrustScore = 0.30*Creator + 0.25*Curator + 0.25*Juror + 0.20*(1000 - RiskScore)`

> 说明：相比原版，降低了 `JurorScore` 权重（0.30→0.25），提高了 `RiskScore` 影响（0.10→0.20）。原因：JurorScore 过高会导致"没当过陪审就难以达标，不达标就当不了陪审"的鸡生蛋困境；而 RiskScore 仅 0.10 权重时，高风险账户的 TrustScore 惩罚幅度仅 ~80 分，威慑不足。

### 2.2 信誉变化（事件驱动，默认强度）

| 事件 | 影响 |
| --- | --- |
| 发帖/提问/回答在 24h 后正常解锁（未被违规裁决） | `CreatorScore +2 ~ +5` |
| 内容被 challenge 且违规成立 | `CreatorScore -20 ~ -80`（按严重度） |
| 内容被 challenge 但不违规 | `CreatorScore +3 ~ +10`（抵抗骚扰式挑战） |
| 你点赞/投票支持的内容最终被判违规 | `CuratorScore -1 ~ -5`（按你权重） |
| 你点赞/投票支持的内容最终被认可 | `CuratorScore +0.5 ~ +2` |
| 作为陪审员，多数裁决与你一致且案件未被上诉推翻 | `JurorScore +3 ~ +10` |
| 作为陪审员，与你相反且你在少数 | `JurorScore -3 ~ -10` |
| 缺席（未 reveal） | `JurorScore -20` + 罚没保证金 |
| 案件上诉推翻你所在的一审多数意见 | `JurorScore -15 ~ -50`（强惩罚，保证严肃投票） |
| 异常资金路径（大额入金后快速提现、关联高风险集群等） | `RiskScore +10 ~ +200`（同时触发提现延迟/限额） |

> **设计说明（增减比例调优）**：正向增长从 `+1~+3` 提升到 `+2~+5`，违规惩罚从 `-30~-120` 缩减到 `-20~-80`。原版中一次违规可抵消 10–120 次正常发帖的信誉积累，极度不对称会导致正常用户因偶尔失误而长期处于低分，抑制参与意愿。调整后一次违规仍等价于 4–40 次正常发帖，依然具有强威慑力，但给了合规用户可恢复的空间。

### 2.3 TrustScore 视觉等级（前端统一标准）

为了让用户快速理解“可信程度”，前端统一采用 5 档颜色映射。内部计算可以继续使用 0–1000，UI 可按 0–100 或 0–1000 展示，但阈值保持一致：

| 档位 | 0–100 分 | 0–1000 分 | 颜色 | UI 表现 |
| --- | --- | --- | --- | --- |
| White | 0–39 | 0–390 | 白色 | 头像白色环 |
| Green | 40–59 | 400–590 | 绿色 | 头像绿色环 |
| Blue | 60–74 | 600–740 | 蓝色 | 头像蓝色环 |
| Purple | 75–89 | 750–890 | 紫色 | 头像紫色环 |
| Orange | 90–100 | 900–1000 | 橙色（主题色） | 头像橙色环 + 发帖卡片橙色边框高亮 |

## 3. 行为规则表（费用/质押/锁定/退回/罚没/分配）

### 3.1 注册与钱包

| 行为 | 费用/质押 | 锁定 | 结果/分配 |
| --- | --- | --- | --- |
| 注册/创建账号 | 0 | 无 | 生成用户与空余额 |
| 绑定钱包地址 | 0 | 无 | 绑定提币地址（可支持多地址白名单） |
| 入金（BTC/闪电/稳定币） | 网络费由用户承担 | 无 | 记账到内部 sat 余额（稳定币按汇率折算） |
| 提现 | `F_withdraw`（可为 0 + 网络费） | 可能延迟 | 见 3.5（提现风控与延迟） |

### 3.2 发帖/提问/回答/评论

| 行为 | 质押/费用 | 锁定 | 正常解锁（无违规裁决） | 违规成立（challenge 成立） |
| --- | --- | --- | --- | --- |
| 发短帖 `note` | `D_post` | 24h | 退回 100% `D_post` | 罚没 50–100% `D_post`（按严重度）+ 信誉下降 + 关闭/删除 |
| 发提问 `question` | `D_question` | 24h | 退回 100% + 可选设置赏金 `Bounty` | 罚没 50–100% + 信誉下降 + 关闭/删除 |
| 回答 `answer` | `D_answer` | 24h | 退回 100% + 可能获得赏金/奖励 | 罚没 50–100% + 信誉下降 + 关闭/删除 |
| 评论 `comment` | `D_comment` | 24h | 退回 100% | 罚没 50–100% + 信誉下降 |

> 备注：严格删垃圾时，**违规罚没比例偏高**。对"明确垃圾/诈骗/恶意引流"可直接 100% 罚没。
>
> **修正说明**：正常解锁从"退回 95–100%"改为"退回 100%"。原来 0–5% 处理费看似微小，但合规用户发 100 条帖子累计被扣 ~1500 sat，形成对正常参与行为的**隐性惩罚税**。平台收入应来自 Boost 广告与违规罚没，而非对正常用户的每次发言抽水——否则与"正常用户可退押金"的核心承诺矛盾。

### 3.3 点赞/投票（强烈建议“有成本”）

| 行为 | 质押 | 锁定 | 正常/正向 | 负向（被证明支持了垃圾） |
| --- | --- | --- | --- | --- |
| 点赞/投票支持内容 | `D_vote` | 24h | 内容在 24h 后正常解锁/未被违规裁决 → 退回 100% `D_vote`，并 `CuratorScore` 小幅上升 | 内容后续被违规裁决 → 扣除 10–50% `D_vote`（按严重度与时间），并 `CuratorScore` 下降 |

> 说明：这条是抗“互赞工会”的关键。没有投票成本，任何奖励机制都会被对刷击穿。

### 3.4 Follow / Unfollow

| 行为 | 费用/质押 | 锁定 | 结果 |
| --- | --- | --- | --- |
| follow / unfollow | 0 | 无 | 仅改变社交关系 |

### 3.5 Chat（私聊/群聊）

| 行为 | 费用/质押 | 锁定 | 结果 |
| --- | --- | --- | --- |
| 发送私聊消息 | 0 | 无 | 即时送达；不适用公开内容质押机制 |
| 创建群聊 | 0 | 无 | 创建者为群主 |
| 发送群聊消息 | 0 | 无 | 群内可见 |

> **Chat 反垃圾策略**（不走质押，走频率限制 + 举报）：
> - 私聊不需要质押（私密通讯不应有经济门槛），但对**非互关用户的私信**设频率上限（例如每日 10 条），防止批量骚扰。
> - 群聊消息不需要质押，但群主/管理员可踢人禁言。
> - 聊天中的举报走简化流程（平台审核或自动检测），**不进入陪审仲裁**（私密消息不适合公开审判）。
> - `RiskScore` 高的用户自动降低私信频率上限。

### 3.6 提现风控（必须项：可提现 + 无 KYC 的安全阀）

> 在无 KYC 前提下，必须用"信誉分层 + 延迟 + 限额"替代传统 KYC 风控。

| 条件（示例） | 单日可提现额度 | 提现延迟 | 备注 |
| --- | --- | --- | --- |
| 新账号（<7天）或 `RiskScore` 高 | 低（例如 50k sat） | 24–72h | 发现异常可延迟更久并触发治理审查 |
| `TrustScore` 中等（400–700） | 中（例如 300k sat） | 6–24h | |
| `TrustScore` 高（>700 且低风险） | 高（例如 2M sat） | 0–6h | 大额仍可随机抽检 |

提现异常触发（任一命中就提高延迟/降低限额）：
- 大额入金后短时间大额提现
- 多账号共享设备/地址簇
- 与已知高风险集群频繁互动/转账

## 4. Challenge（举报/仲裁）机制（严格删垃圾版）

### 4.1 发起 Challenge

| 行为 | 支付 | 锁定 | 结果 |
| --- | --- | --- | --- |
| 对某内容发起 challenge | `F_case`（违规成立退还，不成立不退） + `B_case`（保证金） | 至案件结案 | 进入陪审流程；恶意滥诉会损失 `F_case` + `B_case` |

> **关键设计说明**：`F_case` 在违规成立时退还给挑战者，这是保证"正义举报期望收益为正"的关键。若 `F_case` 任何情况都不退，以发帖质押 300 sat 为例，挑战者即使胜诉也只能获得 ~105 sat 奖励却要永久损失 200 sat 手续费——**赢了也亏钱**，没有理性用户会举报垃圾，整个反垃圾机制将形同虚设。

### 4.2 陪审员资格与抽样

陪审池资格（必须同时满足）：
- `TrustScore ≥ 600`（早期平台可临时降至 500，随用户规模增长逐步提升至 700）
- 账号年龄 ≥ 14 天
- 过去 30 天至少完成 3 次正常投票揭示（不缺席）
- 与当事人存在强关联（关注互粉、交易频繁、同集群）→ **本案排除**

> **修正说明**：原阈值 700 过高。以新用户初始 TrustScore ≈ 550 计算，即使 Creator/Curator 都达到 ~800 分，TrustScore 也才 ~710。在平台早期高信誉用户稀少时，9–15 人的陪审团几乎无法组建。降至 600 并允许早期动态调整，保证治理系统能在冷启动阶段正常运转。

抽样：
- 轻案（普通垃圾/广告/低质）：`N = 9`
- 重案（诈骗/仇恨/未成年人/严重违规）：`N = 15`

### 4.3 投票流程（commit–reveal）

1) **Commit（2h）**：陪审提交投票承诺（不公开选择）
2) **Reveal（4h）**：陪审揭示投票（Yes=违规 / No=不违规）与随机盐

缺席惩罚：
- 未 commit：扣 `B_juror` 的 30% + `JurorScore -10`
- commit 但未 reveal：扣 `B_juror` 的 50% + `JurorScore -20`

> **修正说明**：未 commit 的原罚比例（10% = 30 sat）过轻，陪审员可以以极低成本拒绝参与，导致"被选中但无人投票"的法定人数危机。提高到 30%（90 sat）+ 信誉惩罚后，消极参与的代价与"commit 但不 reveal"形成梯度威慑。

### 4.4 加权投票与裁决阈值（偏向删垃圾）

每个陪审员权重：
- `w = sqrt(TrustScore)`（递减收益，防巨鲸支配）

法定人数：
- `revealed_count ≥ ceil(N * 2/3)`

违规成立阈值（严格删垃圾）：
- 若 `sum(w_yes) / sum(w_revealed) ≥ 0.60` → **违规成立**（删除/关闭/降权）
- 否则 → **不违规**（保留）

> 说明：60% 是“偏严格但不至于一半就删”的平衡点。后续若垃圾攻势强，可将阈值降到 55%（更严格），但误伤会增加，需要依赖上诉兜底。

### 4.5 结算与分配（严格删垃圾：违规时罚没更重）

记 `D_author` 为作者对该内容的质押（例如 `D_post`/`D_question`/`D_answer`）。

#### A) 违规成立（删除/关闭）

- 作者：罚没 `D_author * S`，其中 `S = 0.8 ~ 1.0`（按严重度）
- 挑战者：
  - 退回 `B_case`（全退）
  - 退回 `F_case`（违规成立时退还）
  - 额外奖励 `Reward_challenger = 0.40 * (D_author * S)`
- 陪审多数（投 Yes 的陪审员）：
  - 分享 `0.35 * (D_author * S)`
  - 少数（投 No）可扣 `B_juror` 的 0–20%（按案子置信度）
- 平台/治理池：
  - 获得剩余 `0.25 * (D_author * S)`

> **经济验算（以发帖 300 sat、S=0.9 为例）**：
> - 罚没总额 = 270 sat
> - 挑战者获得：B_case 退回 + F_case 退回 + 0.40×270 = 108 sat 净收益 → **胜诉有正向回报**
> - 陪审多数获得：0.35×270 = 94.5 sat（9 人陪审约 6 人多数，每人 ~16 sat）
> - 平台/治理池：0.25×270 = 67.5 sat
> - 合计分配 = 270 sat ✓

#### B) 不违规（保留）

- 作者：`D_author` 到 24h 正常解锁（退回 100%）
- 挑战者：
  - `F_case` 不退（100 sat 永久损失）
  - `B_case` 罚没 30–60%（150–300 sat，按挑战置信度与历史滥诉情况）
  - 合计损失 250–400 sat → **滥诉代价高昂**
- 陪审多数（投 No 的陪审员）：
  - 分享 `F_case`（100 sat）+ 可额外分享 `B_case` 罚没的 20%
- 平台/治理池：
  - 获得 `B_case` 剩余罚没部分

> **设计验证**：正义举报期望收益为正（激励举报垃圾），恶意滥诉期望收益为负（遏制职业骚扰），两条约束同时满足。

## 5. 上诉（Appeal）机制（严格删垃圾：推翻更难）

### 5.1 上诉触发

仅当一审裁决完成后，24h 内可上诉。

- 上诉方支付：`F_appeal`（不退，建议为 `2*F_case` = 200 sat）+ `B_appeal`（保证金，建议为 `2*B_case` = 1000 sat）
- 二审陪审：`N = 21`（早期可临时降至 15）

> **修正说明**：原方案 `F_appeal=3*F_case + B_appeal=3*B_case` 总上诉成本 = 600+2400 = 3000 sat，远超内容质押本身（300–500 sat），使得普通用户实质上无力上诉，二审机制形同虚设。降低至 2 倍后总成本 = 200+1000 = 1200 sat，仍高于一审挑战成本（起到筛选作用），但不至于完全阻断救济通道。同理，31 人二审陪审团在早期几乎不可能凑齐，调整为 21（早期可降至 15）。

### 5.2 二审裁决与“推翻门槛”

二审仍用加权投票，但推翻一审需要更强共识：

- 若一审为“违规成立”，二审要推翻（改为不违规）需满足：`sum(w_no)/sum(w_revealed) ≥ 0.70`
- 若一审为“不违规”，二审要推翻（改为违规）需满足：`sum(w_yes)/sum(w_revealed) ≥ 0.70`

> 这样在严格删垃圾取向下，**一审删掉的内容更难被救回来**，除非二审高度一致认为误判。

### 5.3 上诉结算

上诉失败（未推翻）：
- 上诉方损失大部分 `B_appeal`（例如 60–90%）
- 二审多数分享 `F_appeal` + 罚没的一部分

上诉成功（推翻）：
- 上诉方返还 `B_appeal`（全退）+ 获得部分奖励（来自一审罚没池/平台治理池）
- 一审多数陪审：`JurorScore` 重罚（见 2.2）+ 可扣一部分其 `B_juror`（建立强约束）

## 6. Boost / 广告（平台核心收入）

### 6.1 广告与普通内容的统一

任何 `Post` 都可设置为广告/推广内容（标记 `is_ad = true`），但仍需遵守内容政策与 challenge 机制。

### 6.2 Boost 购买

广告主为某条内容设置：
- `budget_sat`：总预算（从内部余额扣除）
- `targeting`：语言/地区/频道/标签（可选）
- `freq_cap`：频控（每用户每 24h 最多触达次数）

计费建议（简化可落地版）：
- 每次触达消耗：`cost = c_base * M(SI) * AudienceQuality`
  - `AudienceQuality`：疑似真人更高、疑似机器人更低甚至不投放
  - 负反馈（隐藏/拉黑/挑战成功）会提高 `c_base` 或触发熔断

### 6.3 Boost 收入分配（平台最多，但建议保留“治理成本”极小比例）

平台拿绝大部分收入。为了让仲裁系统长期可靠（陪审有激励、治理池有预算），建议最低保留：

- **平台收入**：95–98%
- **治理池（用于陪审奖励/反垃圾运营）**：2–5%
- **受众返利/基金**：可为 0%

> 治理池建议保留，否则所有治理成本都得从 challenge 费里收，会抬高用户参与门槛。
>
> **冷启动阶段治理池资金来源**：在 Boost 收入尚未规模化之前（Phase 1 早期），治理池资金来源有限。建议平台预留一笔初始运营资金（例如 10M sat ≈ 0.1 BTC）作为治理池种子资金，用于支付早期陪审奖励。这笔资金可视为平台获客/运营成本的一部分。

## 7. 内容政策与“严格删垃圾”的执行建议（非数值规则，但很关键）

为了让陪审可执行，必须定义“可操作的违规条款”，否则陪审将变成主观吵架。

建议至少包含：
- 垃圾广告/引流（重复、批量、低信息密度）
- 诈骗与冒充（钱包地址诱导、虚假项目、钓鱼链接）
- 恶意软件/钓鱼站
- 仇恨/暴力/未成年人相关违法内容
- 恶意骚扰与人身攻击

严格删垃圾策略下建议：
- 对“诈骗/钓鱼/恶意引流”设为重案（N=15，罚没更高）
- 对“重复模板广告”允许更快处理（轻案即可删）

## 8. MVP 实现清单（工程落地参考）

最小可上线闭环（建议优先级）：
- 主导航架构：Feed / Following / 发布 / Chat / 我的（底部 Tab Bar + 中央发布按钮）
- 内部 sat 账本：余额、冻结余额（锁定质押）、解锁/罚没记账
- 发帖/提问/回答/评论：质押锁定与 24h 解锁
- 点赞/投票：质押锁定与事后结算（与裁决联动）
- Chat：私聊/群聊（基础文字+图片，频率限制反垃圾）
- challenge：案件创建、陪审抽样、密封投票（Phase 2 升级为 commit–reveal）、裁决与结算
- 信誉系统：四维分与合成分；用于费用折扣与陪审资格
- 提现：分层限额 + 延迟 + 风控触发
- boost：预算扣费 + 频控 + 分配（至少含平台与治理池）
